What to do when something breaks

                
    
                    
  1. Status dot red above or BetterStack alert: check the raw /health response for the error. If DB is unreachable, check Railway → Postgres service status. If the whole API is down, check Railway → web service deploys.
    2. 
                    
  2. Sentry error spike: open Sentry → sort by "most users affected" → read the top issue. Each event is tagged with the git SHA of the release that caused it.
    3. 
                    
  3. Mobile crash on new build: Sentry → mobile → filter by latest release. If stack traces look minified (e.g. index.bundle:1:24875), source-map upload failed on EAS build — check the build log.
    4. 
                    
  4. Funnel dip (signups drop) without an error spike: not a code issue. Check PostHog → break down by country / OS / app version (the 4/20 pattern).
    5. 
                

                
Full plan: docs/monitoring-strategy.md in the repo.
