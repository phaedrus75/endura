# Endura — v1.0.3 (build 21)

**Released to TestFlight:** 2026-04-27
**Platforms:** iOS (App Store / TestFlight)
**Previous build:** v1.0.2 (build 20)

---

## What's new for users

### Send feedback from inside the app
- New chat icon (💬) on the Home screen, just left of your profile picture.
- Tap it to open a feedback sheet where you can pick a category — **Bug**, **Idea**, **Question**, or **Love** — write a message, and (optionally) **attach up to 4 screenshots**.
- Email is pre-filled when you're signed in, so we can reply.
- App version, OS, device model and the screen you were on are auto-attached for triage — you don't need to type any of it.
- Everything goes straight into the support queue.

### Friends leaderboard now shows everyone correctly
- Fixed: high study-time friends would sometimes drop off the friends leaderboard, and the user themselves could appear at the bottom even when they had the highest hours.
- Cause was server-side: the deployed query truncated to 20 rows *before* sorting by minutes, and legacy `NULL` values crashed the sort.
- The leaderboard now fetches the full friend set, sorts in code, and only then trims to the leaderboard size. Anyone you're friends with who has logged any minutes will show in their correct position.
- Added a defensive UI fallback so you'll always see your own row, even if the server response is unexpectedly missing it.

### Egg hatching is now glitch-proof
- Previously: if you didn't open the app right when a timer finished, the "ready to hatch" egg could disappear and your progress was lost.
- Now: the next time you open the app, the egg-ready prompt pops back up exactly where you left off. Nothing is lost.

### "Timer done" notification
- The moment a study timer hits 00:00, you'll get a local notification: *"Your timer is done! Tap to hatch egg 🥚"*. Works even if the app is in the background.
- These local notifications are now also tracked in our internal admin dashboard so we can see they actually fired (no more "did the notification work?" guessing).

### Tap the egg to start studying
- The egg illustration on the Home screen (above the **STUDY TO HATCH ME** button) is now tappable — it takes you straight to the timer.

### Better study-group leaving
- Members added to a study group now have a clear option to leave it.

### Onboarding tweaks
- The exit-confirmation popup now says *"your animal will never hatch"* instead of revealing which animal it would have been — keeping the surprise intact.
- Profile picture is no longer required during sign-up; falls back to your initials and can be changed any time from the Profile screen.

---

## Under the hood (no user-visible UI, but worth knowing)

- New backend endpoint `POST /feedback/attachments` for image uploads (auth-required, 5 MB cap, magic-byte validated, reuses the existing `Upload` model).
- `user_feedback.attachment_urls` column added (Alembic migration `q1m2n3o45p16`). Legacy `screenshot_url` is auto-populated with the first attachment for back-compat with older admin views.
- Admin dashboard: feedback detail panel now shows all attachments as a thumbnail grid (each clickable for full-size).
- Sanitization: any `attachment_urls` not pointing at our own `/uploads/` host are silently dropped server-side.
- Backend regression tests added for: friends-leaderboard NULL-safety, friends-leaderboard truncation order, attachment upload (auth/non-image rejection, end-to-end submit-with-attachments, external-URL rejection), local push delivery + dedup.
- Database migration `p0l1m2n34o15` backfills any legacy `NULL` user-stat columns to `0` and locks them `NOT NULL` going forward.
