**Released to TestFlight:** 2026-05-03
**Platforms:** iOS (App Store / TestFlight) + Android (Play Store)
**Previous shipped build:** v1.0.3 (build 25)

> uploaded to TestFlight and is what's being promoted to App Store.

## What's new for users

### Sessions never disappear, even if your phone dies mid-timer
This was the headline fix for this release. Before 1.0.5, a small but
real fraction of study sessions were silently lost: you'd start a
25-minute timer, the OS would suspend or kill Endura while it was in
the background, and when you came back the timer was gone — no coins,
no streak credit, no animal hatched.

In 1.0.5 we fixed this in three places at once:

1. **The app now records the moment your timer starts**, not just when
   it ends. So even if Endura is killed during the session, when you
   reopen it the next time we know what you were doing.
2. **When you reopen, we automatically resume** — either continuing
   the timer if it's still running, or finishing the session and
   giving you your reward if the time's already up.
3. **Even if you never reopen Endura at all**, the server now
   auto-credits you for sessions where the timer should have finished.
   So if you study for 25 minutes, walk out of the room and forget
   about Endura, you'll *still* see your coins and streak the next
   time you open the app.

The combined effect: study sessions essentially stop disappearing.

### Better resilience to bad network conditions
- If your wifi drops at the exact moment a timer ends, Endura no
  longer silently swallows the session. The next time you come back
  online, it automatically retries and credits you correctly.
- A small "save in progress" indicator surfaces if something is
  unusual.

### New first-launch experience for everyone
- A short visual walkthrough now plays **before** you're asked to sign
  up — so first-timers see what Endura actually does (study timer →
  hatch animal → grow sanctuary) before being asked for an email.
- Returning users on previous builds (who saw the older auth-first
  flow) are silently migrated on cold launch — no action required, no
  data loss; the walkthrough simply replaces the legacy auth-first
  screen on next open.
- This came out of an A/B test that ran from 1 May: the
  walkthrough-before-auth variant produced **+58% activation** (3+
  completed sessions in 72 h) versus the original auth-first flow.
  v1.0.5 promotes the winning variant to 100% of users.

## Under the hood (no user-visible UI, but worth knowing)

- New backend endpoint `POST /sessions/start` records study intent at
  the moment the timer begins (`completed_at IS NULL`). Pairs with the
  existing `POST /sessions/{id}/complete` for the new client
  handshake, while the legacy `POST /sessions` one-shot path is
  preserved as a fallback for older builds in the wild.
- New `study_sessions.auto_completed_at` column (Alembic migration
  `y2z3a4b56c27` + boot-time safety-net ALTER) marks rows that were
  finalised by the new server-side reaper rather than by an explicit
  client call. Used by `/admin/sessions/incomplete` to distinguish
  auto-credited sessions in admin dashboards.
- New crud helper `reap_stale_sessions()` finds rows where
  `started_at + duration_minutes + 30m grace < now()` and finalises
  them via the same `_finalize_session()` path real completions use
  (so coins / streak / totals all match exactly). Idempotent on the
  `auto_completed_at` marker, bounded by `max_age_hours=168` so we
  don't retroactively credit ancient dev artefacts.
- APScheduler runs the reaper every 15 minutes inside the Railway
  process; also exposed as `POST /admin/sessions/reap-stale` for
  manual one-shot catch-up runs from the admin dashboard.
- Frontend `TimerScreen.tsx`: full state machine for resilient
  sessions, recovery effect on user load, AsyncStorage-backed
  `ACTIVE_TIMER_KEY` and `PENDING_HATCH_KEY` per-user, Sentry
  breadcrumbs around every start / complete / fallback / retry /
  recover / abandon transition. iOS `'inactive'` transitions
  (Notification Center swipe, app-switcher peek, system permission
  prompts) no longer trigger the destructive "Abandon Egg" alert
  spuriously.
- Recovery path now fires `Analytics.sessionCompleted` and
  `Analytics.eggHatched` so PostHog accurately credits sessions that
  were finalised after a JS-context kill (previously these looked
  like silent drop-offs in the funnel even when the user got their
  reward).
- `Analytics.sessionStarted` is now fired *after* `persistActiveTimer`
  succeeds, so a kill in the millisecond gap between `Start` press and
  state being durable doesn't produce a false-positive started-but-
  never-completed funnel row.
- 18 new backend tests in `tests/api/test_sessions.py` and
  `tests/api/test_session_failure_modes.py` cover: start/complete
  handshake, ownership 404, double-complete 409, history exclusion,
  admin listing, reaper credit / grace-window / idempotency /
  already-completed safety / max-age horizon, stranded-row
  visibility, long-idle recovery, concurrent double-start. 251/251
  backend + 13/13 frontend tests green.

### Telemetry & admin fixes also in build 32 (post-1.0.4 cut)

- **App-version capture moved off the push-notification path.**
  Previously `users.app_version` was only ever populated by the
  push-token registration flow, which meant any user who declined push
  permission, ran on a simulator, or hit a transient Expo push token
  failure was missing a version on their row. We now stamp
  `X-App-Version` + `X-App-Build` headers on every authenticated API
  call and update the user row opportunistically server-side
  (`backend/auth.py::_capture_app_version_from_headers`). Backfilled
  with 5 new tests in `test_app_version_capture.py`.
- **Admin product-test funnel: `cohort_started_at` filter added.**
  The onboarding A/B funnel in the admin dashboard was over-counting
  the v1 arm by including users who signed up before the experiment
  binary actually shipped to TestFlight (they were assigned a variant
  on first opening the new build, not at signup). New
  `product_tests.cohort_started_at` column (Alembic
  `z3a4b5c67d28` + boot-time safety-net ALTER) lets us pin the cohort
  floor to a specific date; `_funnel_arm_counts` now filters
  `users.created_at >= cohort_started_at`. Edit panel in the admin
  dashboard exposes a date input next to the PostHog insight URL.
- **Onboarding A/B promotion path:** `App.tsx` now hardcodes v2 as the
  assigned variant for new installs and silently migrates any
  `SecureStore` row carrying `'v1'` over to `'v2'` on cold launch
  (analytics source `promoted_from_v1` so we can audit how many
  legacy assignments were rewritten in the first 7 days post-ship).
- **Build hygiene:** added the untracked `scaffolding/` directory to
  `.gitignore` after EAS rejected a build for inconsistent filename
  casing in files that were being included in the EAS tarball but not
  visible to local case-insensitive HFS+ scans.

### What's intentionally NOT in 1.0.5

- **Sign in with Apple + Sign in with Google buttons** — code is in
  the codebase (backend endpoints, JWKS verification, merge logic,
  `oauthLogin.ts` helper) but the UI is gated until Apple Developer
  capabilities + Google Cloud client IDs are provisioned and tested.
  Full re-enable playbook in `docs/oauth-setup.md`.

What's new in v1.0.5:

• Study sessions stop disappearing — if your phone dies, the OS kills
  Endura in the background, or you forget to reopen the app, your
  session is now automatically saved and credited. The bug where you'd
  start a 25-minute timer, come back, and find no coins / no streak is
  fixed end-to-end.

• Better behaviour when your network drops — if wifi cuts out at the
  exact moment a timer ends, Endura now retries automatically next
  launch instead of silently losing the session.

• New first-launch walkthrough — first-time users now see a short
  visual intro to Endura before being asked to create an account. The
  A/B test that drove this change showed +58% activation (3+ sessions
  in 72 h) versus the previous auth-first flow, so it's now the
  default for everyone.

What to focus on:
1. Start a 25-minute Test Timer (Settings → Test Timer ON makes each
   "minute" 1 second so you can iterate fast). Force-quit Endura from
   the app switcher mid-timer. Reopen — the session should either
   resume (if still in window) or auto-complete with your reward (if
   the timer would have finished).
2. Same as 1 but DON'T reopen Endura. Wait ~30 minutes, then open it.
   The session should already have been auto-credited by the server
   reaper — your coin balance / streak should reflect it.
3. Toggle airplane mode ON right before a timer ends. Toggle OFF a
   minute later. Confirm the session is credited (may take a moment).
4. (Fresh install only) Delete + reinstall the app. On first open you
   should see the walkthrough screens before the sign-up screen — not
   the sign-up screen first. Returning testers will be silently
   migrated to the same flow on next cold launch.

• Reaper-credited sessions credit coins + streak correctly but don't
  currently run the egg-hatching animation (the animal you wanted to

  fixed in 1.0.6.
• Sign in with Apple / Google removed in this build vs 1.0.4 — coming
  back in 1.0.6 once provider config is complete.
